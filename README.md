## Użyte polecenia
Do tej pory w projekcie użyliśmy następujących poleceń:
* `git init` - inicjalizacja lokalnego repozytorium
* `git status` - sprawdzanie stanu projektu
* `git add` - dodawanie plików do poczekalni (staging area)
* `git commit` - trwałe zapisywanie zmian w historii
* `git push` - wysyłanie kodu na serwer GitHub
* `git checkout -b` - tworzenie nowej gałęzi i przełączanie się na nią
